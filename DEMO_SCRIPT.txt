[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the system architecture. When a user sends a message, it enters the workflow starting at the START node. The first stop is the Security Checkpoint. This node performs real-time PII scrubbing and scans for prompt injections or self-harm keywords. 

If there is a safety warning, it routes directly to the Security Violation Handler. If the input is safe, the Intent Router checks if the user is asking for nutrition plans. If so, a Human-in-the-Loop pause is triggered using a dedicated allergy check to ask for dietary restrictions. 

Once resolved, the Orchestrator agent coordinates with two specialized sub-agents: the Health Assessor and the Diet Planner. These sub-agents query our Model Context Protocol, or MCP, server to log daily symptoms, record meal plans, or fetch history, before passing the formatted result to the user.

[3:00 - SECURITY & MCP]
This secure routing is backed by a structured audit logging system. Every decision is written to a local JSON file, cataloging INFO, WARNING, and CRITICAL events. Furthermore, the MCP server manages our local storage, keeping secure records of the teen's diet logs, symptom history, and reminders, which can be queried dynamically by the sub-agents.
